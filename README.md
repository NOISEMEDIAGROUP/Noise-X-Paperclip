## Paperclip 是什么？

# 零人力公司的开源编排系统

**如果说 OpenClaw 是一个_员工_，那么 Paperclip 就是整个_公司_**

Paperclip 是一个 Node.js 服务器和 React 用户界面，用于编排一个 AI 智能体团队来运营企业。接入你自己的智能体，分配目标，并在一个仪表板中跟踪智能体的工作和成本。

它看起来像一个任务管理器——但底层具备组织架构图、预算、治理、目标对齐和智能体协调等功能。

**管理业务目标，而非拉取请求。**

|        | 步骤            | 示例                                                            |
| ------ | --------------- | ------------------------------------------------------------------ |
| **01** | 定义目标 | _"打造排名第一的 AI 笔记应用，实现 100 万美元月经常性收入。"_                    |
| **02** | 组建团队   | CEO、CTO、工程师、设计师、市场人员——任何机器人，任何供应商。 |
| **03** | 审批并运行 | 审查策略。设定预算。启动运行。通过仪表板监控。  |

<br/>

> **即将推出：Clipmart** — 一键下载并运行整个公司。浏览预构建的公司模板——完整的组织架构、智能体配置和技能——几秒钟内即可导入到你的 Paperclip 实例中。

## Paperclip 适合你，如果

- ✅ 你想构建**自主运营的 AI 公司**
- ✅ 你需要**协调多种不同的智能体**（OpenClaw、Codex、Claude、Cursor）朝着共同目标工作
- ✅ 你同时打开了 **20 个 Claude Code 终端**，已经分不清每个在做什么
- ✅ 你希望智能体 **全天候自主运行**，但仍然想审查工作并在需要时介入
- ✅ 你想**监控成本**并执行预算
- ✅ 你想要一个管理智能体的流程，**像使用任务管理器一样简单**
- ✅ 你想**在手机上**管理你的自主业务

## Paperclip 解决的问题

| 没有 Paperclip                                                                                                                     | 有了 Paperclip                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| ❌ 你打开了 20 个 Claude Code 标签页，无法追踪每个在做什么。重启后一切丢失。                              | ✅ 任务基于工单，对话以线程组织，会话在重启后依然保留。                                                |
| ❌ 你手动从多个地方收集上下文，提醒机器人你到底在做什么。                                     | ✅ 上下文从任务向上贯穿项目和公司目标——你的智能体始终知道做什么以及为什么。                  |
| ❌ 智能体配置文件夹杂乱无章，你在重新发明任务管理、通信和智能体间的协调。 | ✅ Paperclip 开箱即用地提供组织架构图、工单系统、委派和治理——让你运营公司，而不是管理一堆脚本。 |
| ❌ 失控的循环浪费数百美元的 token，在你察觉之前就耗尽了配额。                           | ✅ 成本追踪显示 token 预算，并在智能体超支时进行限流。管理层通过预算进行优先级排序。                    |
| ❌ 你有定期任务（客户支持、社交媒体、报告），必须记着手动启动。                        | ✅ 心跳机制按计划处理常规工作。管理层负责监督。                                                                |
| ❌ 你有一个想法，必须找到仓库，启动 Claude Code，保持标签页打开，全程守着。                                | ✅ 在 Paperclip 中添加一个任务。你的编程智能体会持续工作直到完成。管理层审查他们的工作。                              |

## Paperclip 的独特之处

Paperclip 正确处理了复杂的编排细节。

|                                   |                                                                                                               |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| **原子化执行。**             | 任务签出和预算执行是原子化的，杜绝重复工作和失控开支。                      |
| **持久化智能体状态。**       | 智能体在心跳之间恢复相同的任务上下文，而非从头开始。                     |
| **运行时技能注入。**      | 智能体可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。                      |
| **可回滚的治理机制。**     | 审批关卡被强制执行，配置变更有版本记录，错误的变更可以安全回滚。        |
| **目标感知执行。**         | 任务携带完整的目标血统，智能体始终能看到"为什么"，而不仅仅是标题。                        |
| **可移植的公司模板。**   | 导出/导入组织、智能体和技能，带有密钥清除和冲突处理功能。                          |
| **真正的多公司隔离。** | 每个实体都有公司作用域，一次部署可以运行多家公司，各自拥有独立的数据和审计记录。 |

## Paperclip 不是什么

|                              |                                                                                                                      |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **不是聊天机器人。**           | 智能体有工作职责，而非聊天窗口。                                                                                  |
| **不是智能体框架。**  | 我们不告诉你如何构建智能体。我们告诉你如何运营一家由智能体组成的公司。                                |
| **不是工作流构建器。**  | 没有拖拽式流水线。Paperclip 建模的是公司——包含组织架构图、目标、预算和治理。            |
| **不是提示词管理器。**    | 智能体自带提示词、模型和运行时。Paperclip 管理的是它们所在的组织。               |
| **不是单智能体工具。** | 这是为团队设计的。如果你只有一个智能体，可能不需要 Paperclip。如果你有二十个——那你绝对需要。 |
| **不是代码审查工具。**  | Paperclip 编排工作，而非拉取请求。代码审查流程请自行配备。                                       |

## 快速开始

开源。自托管。无需 Paperclip 账号。

```bash
npx paperclipai onboard --yes
```

或手动操作：

```bash
git clone https://github.com/paperclipai/paperclip.git
cd paperclip
pnpm install
pnpm dev
```

这将在 `http://localhost:3100` 启动 API 服务器。内嵌的 PostgreSQL 数据库会自动创建——无需任何设置。

> **系统要求：** Node.js 20+, pnpm 9.15+

## 常见问题

**典型的部署方式是怎样的？**
在本地，一个 Node.js 进程管理内嵌的 Postgres 和本地文件存储。在生产环境中，将其指向你自己的 Postgres 并按你喜欢的方式部署。配置项目、智能体和目标——剩下的交给智能体处理。

如果你是独立创业者，可以使用 Tailscale 在外出时访问 Paperclip。之后在需要时可以部署到例如 Vercel。

**可以运行多家公司吗？**
可以。单次部署可以运行无限数量的公司，数据完全隔离。

**Paperclip 与 OpenClaw 或 Claude Code 等智能体有什么区别？**
Paperclip _使用_这些智能体。它将它们编排成一家公司——具备组织架构图、预算、目标、治理和问责机制。

**为什么我应该使用 Paperclip，而不是直接让 OpenClaw 对接 Asana 或 Trello？**
智能体编排在协调谁签出了哪些工作、如何维护会话、监控成本、建立治理等方面有诸多微妙之处——Paperclip 为你解决这些问题。

（自带工单系统功能已在路线图中）

**智能体是持续运行的吗？**
默认情况下，智能体按计划的心跳和基于事件的触发器（任务分配、@提及）运行。你也可以接入像 OpenClaw 这样的持续运行智能体。你提供智能体，Paperclip 负责协调。

## 开发

```bash
pnpm dev              # 完整开发模式（API + UI，监听模式）
pnpm dev:once         # 完整开发模式，不监听文件变化
pnpm dev:server       # 仅服务器
pnpm build            # 构建全部
pnpm typecheck        # 类型检查
pnpm test:run         # 运行测试
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 应用迁移
```

完整开发指南请参阅 [doc/DEVELOPING.md](doc/DEVELOPING.md)。
